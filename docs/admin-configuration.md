# Admin Configuration

This guide covers all site-level administration settings for the AI Assisted Feedback plugin.

## Accessing the Settings

Navigate to:

**Site Administration → Plugins → Activity modules → Assignment → Feedback plugins
→ AI Assisted Feedback**

## Settings Reference

### Enabled by Default

**Setting key:** `assignfeedback_aif/default`

| Property | Value |
|----------|-------|
| Type | Checkbox |
| Default | Disabled |

When enabled, the AI Assisted Feedback plugin will be automatically activated for every new
assignment created on the site. Teachers can still disable it per assignment.

> **Recommendation:** Leave disabled on large sites. Let teachers opt-in per assignment to
> avoid unexpected AI costs.

---

### Default Prompt

**Setting key:** `assignfeedback_aif/prompt`

| Property | Value |
|----------|-------|
| Type | Textarea |
| Default | "Analyse the grammar in this text" |

The default prompt that is pre-filled when a teacher creates a new assignment with the
AI feedback plugin enabled. Teachers can override this per assignment.

The prompt is included in the `{{prompt}}` placeholder of the prompt template (see below).

**Examples:**
- `Analyse the grammar and structure of this essay.`
- `Provide feedback on code quality, readability, and correctness.`
- `Evaluate this lab report for scientific accuracy and clarity.`

---

### AI Backend

**Setting key:** `assignfeedback_aif/backend`

| Property | Value |
|----------|-------|
| Type | Select dropdown |
| Default | `core_ai_subsystem` |
| Options | Core AI Subsystem, Local AI manager |

Choose which AI system processes the requests:

- **Core AI Subsystem** — Moodle's built-in AI framework (available since Moodle 4.5).
  Configure providers in *Site Administration → AI → AI providers*.
- **Local AI manager** — The `local_ai_manager` plugin, offering advanced features like
  usage quotas, role-based configuration, and purpose-specific AI tools.

See [AI Backends](ai-backends.md) for detailed setup instructions for each backend.

---

### AI Purpose

**Setting key:** `assignfeedback_aif/purpose`

| Property | Value |
|----------|-------|
| Type | Text input |
| Default | `feedback` |
| Validation | PARAM_ALPHANUMEXT |

**Only relevant when using the local_ai_manager backend.**

The purpose identifier tells local_ai_manager which AI tool configuration to use. This allows
administrators to set up different AI providers, models, or quotas for different use cases.

**Common values:**
- `feedback` — General feedback generation
- `translate` — Text translation (used internally for disclaimer translation)
- `summarise` — Text summarisation
- `itt` — Image-to-text analysis (used automatically for image submissions)

The purpose must be configured in the local_ai_manager admin settings to map to an actual
AI provider.

### Prompt Template

**Setting key:** `assignfeedback_aif/prompttemplate`

| Property | Value |
|----------|-------|
| Type | Textarea (80 cols × 15 rows) |
| Default | (structured template — see below) |

The prompt template defines the structure of every AI request. It uses placeholders that are
replaced with actual values at generation time.

**Available Placeholders:**

| Placeholder | Replaced With |
|-------------|--------------|
| `{{submission}}` | The student's submission text (online text + extracted file content) |
| `{{rubric}}` | Rubric criteria and level definitions (empty if no rubric configured) |
| `{{prompt}}` | The teacher's per-assignment prompt |
| `{{assignmentname}}` | The assignment's name |
| `{{language}}` | The current user's language (e.g., "English", "Deutsch") |

**Default Template:**

```
=== ROLE ===
You are an experienced teacher providing constructive feedback on student submissions.

=== ASSIGNMENT ===
{{assignmentname}}

=== GRADING CRITERIA ===
{{rubric}}

=== TEACHER INSTRUCTIONS ===
{{prompt}}

=== STUDENT SUBMISSION ===
{{submission}}

=== OUTPUT INSTRUCTIONS ===
Provide detailed, constructive feedback that helps the student improve.
Focus on both strengths and areas for improvement.
Be encouraging but honest.

=== LANGUAGE ===
Respond in {{language}}.
```

See [Prompt Template System](prompt-template-system.md) for advanced customization examples.

### Disclaimer

**Setting key:** `assignfeedback_aif/disclaimer`

| Property | Value |
|----------|-------|
| Type | Text input |
| Default | "(This feedback was generated by an AI system and reviewed by your teacher.)" |

This text is automatically appended to every AI-generated feedback. It informs students that
the feedback was produced by an AI system.

**To disable:** Leave the field empty. No disclaimer will be appended.

**Formatting:** The disclaimer is wrapped in a `<p><em>...</em></p>` HTML tag when appended.

---

### Translate Disclaimer

**Setting key:** `assignfeedback_aif/translatedisclaimer`

| Property | Value |
|----------|-------|
| Type | Checkbox |
| Default | Enabled |

When enabled, the disclaimer text is automatically translated to the student's preferred
language using the configured AI backend.

**How it works:**
1. The plugin detects the user's current Moodle language.
2. If the language is not English, the disclaimer is sent to the AI for translation.
3. Translations are cached in the `translations` application cache to avoid repeated AI calls.
4. If translation fails, the original English disclaimer is used as fallback.

**Cache:** Translations are stored with a key format of `lang_<languagecode>` (e.g., `lang_de`)
in the `assignfeedback_aif/translations` application cache.

> **Note:** Each unique language requires one AI call for translation. On a site with many
> languages, consider whether the translation cost is acceptable.

---

### Practice Mode Disclaimer

**Setting key:** `assignfeedback_aif/practicedisclaimer`

| Property | Value |
|----------|-------|
| Type | Text input |
| Default | "(This feedback was generated by an AI system for self-study purposes. It has NOT been reviewed or approved by a teacher. Use it as a learning aid, but be aware that AI feedback may contain errors.)" |

This disclaimer is used instead of the regular disclaimer when the assignment is in
**practice mode** (autogenerate enabled + marking workflow disabled).

In practice mode, students see AI feedback immediately without teacher review. The practice
disclaimer clearly communicates this to students.

**When is which disclaimer used?**

| Scenario | Disclaimer Used |
|----------|----------------|
| Autogenerate ON + Marking workflow OFF | **Practice disclaimer** |
| Autogenerate ON + Marking workflow ON | Regular disclaimer |
| Autogenerate OFF (manual/batch) | Regular disclaimer |

> **Recommendation:** Customise this text to match your institution's AI usage policy.
> Emphasise that the feedback has not been reviewed by a teacher and should be treated
> as a learning aid, not as an authoritative assessment.

**To use the same disclaimer for both modes:** Set the practice disclaimer to the same
text as the regular disclaimer.

## Configuration Checklist

A quick checklist to ensure the plugin is properly configured:

1. ☐ **AI Provider configured** — At least one AI provider must be set up:
   - Core AI: *Site Administration → AI → AI providers*
   - local_ai_manager: Configure providers and purposes in the plugin settings
2. ☐ **Backend selected** — Choose the correct backend in the plugin settings
3. ☐ **Prompt template reviewed** — Ensure the default template suits your institution's needs
4. ☐ **Disclaimer customised** — Adjust both the regular and practice disclaimers for your
   institution's AI policies
5. ☐ **Document converter** (optional) — If you want file submission analysis, configure a
   document converter in *Site Administration → Plugins → Document converters*
6. ☐ **Cron running** — Ensure cron runs frequently (every minute recommended) for timely
   feedback generation

## Troubleshooting

### No feedback is generated

1. **Check cron:** Feedback is generated asynchronously. Ensure cron is running.
   Check *Site Administration → Server → Scheduled tasks* for task status.
2. **Check AI provider:** Verify the AI provider is correctly configured and accessible.
3. **Check backend setting:** Ensure the selected backend matches your installed AI system.
4. **Check task logs:** Look at *Site Administration → Server → Task logs* for error messages.

### Error: "Unable to retrieve feedback. AI system configuration might be wrong"

This error indicates the AI backend returned no usable response. Check:
- The AI provider API key is valid and not expired.
- The AI provider service is accessible from the Moodle server.
- For local_ai_manager: The configured purpose has a valid provider assigned.

### Disclaimer not translating

1. Verify "Translate disclaimer" is enabled in settings.
2. Check if the user's language is different from English.
3. Look for translation errors in the task logs.
4. Try purging the translations cache: *Site Administration → Development → Purge all caches*.
